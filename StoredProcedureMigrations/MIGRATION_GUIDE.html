How to run (Production, by hand)

All scripts are CREATE OR ALTER / idempotent and safe to re-run. Replace SERVER with the production instance.

# Run each phase in order with sqlcmd (-b = stop on error). Production = TradeNetDB.
SET QUOTED_IDENTIFIER ON      # (sqlcmd sets these ON by default; pymssql does NOT)

# Phase 1 - indexed views (run BEFORE the procedures)
for f in StoredProcedureMigrations/Views/*.sql;            do sqlcmd -S SERVER -d TradeNetDB -i "$f" -b; done
# Phase 2 - consolidated production indexes
sqlcmd -S SERVER -d TradeNetDB -i StoredProcedureMigrations/IndexedMigrations/TradeNetReportIndexes_Production.sql -b
# Phase 3 - stored procedures
for f in StoredProcedureMigrations/*.sql;                  do sqlcmd -S SERVER -d TradeNetDB -i "$f" -b; done
SSMS / Azure Data Studio: connect to the production TradeNetDB, then open & Execute each file in the Phase 1 → 2 → 3 order above. The index script prints its own preflight and aborts if not connected to TradeNetDB.
